Lilith 0.9.39

Download Now!
(Released: 2010-05-12)

Changes

  • Added some error messages.
  • Added "recent files" menu.
    This was requested by Jeff Jensen (jeffjensen).
    (Ticket 83)
  • lilith.bat does not strictly require a %JAVA_HOME% environment variable anymore.
    This was requested by Gareth Doutch (gdoutch).
    (Ticket 80)
  • Added special icons for views opened from a file, different ones for refreshing (Lilith) and non-refreshing (imported) ones.
    They arguably suck a little bit. ;)
    (Ticket 48)
  • Added automatic refreshing of opened Lilith files. This should work but isn't tested very thoroughly, yet.
    This was requested by Jeff Jensen (jeffjensen).
    (Ticket 84)
  • Added cat (-c) and tail (-t, -f) functionality to Lilith so it's possible to print entries of binary Lilith log-files to the console. Number of printed entries can be configured using -n. The format string of for the event can be given at runtime.
    Unfortunately, this is only possible with Logback-Classic, not Logback-Access.
    The reason for this shortcoming is LBACCESS-12.

    The following parts of LoggingEvent aren't supported yet:
    • LoggerContextVO
    • IThrowableProxy
    • Marker
  • Removed LogbackClassic appender from default logback.xml of Lilith to prevent warning during startup. Those would be especially annoying in case of cat or tail.
    This is a workaround for LBCLASSIC-156.
  • Changed autocomplete of find combos from uber-annoying case-insensitive swingx to much less annoying case-sensitive Glazed Lists.
    This annoyed Alfred and me.
  • Neither "Window" menu nor status bar did update properly if "Automatically open new views on connection." was deselected.
    This bug was found by Joe.
    (Ticket 82)
  • Changed the license of all files previously licensed as LGPLv3 to both LGPLv3 and ASLv2 instead. Use whichever license suits you better.
    This was requested by Ekkehard Gentz.
  • Added option to maximize internal frames by default. This is actually very good idea!
    This was suggested by snstanton.
    (Ticket 77)
  • Added contextName and/or applicationIdentifier to the title of the frames.
    This was suggested by Alfred & Joe.
  • It wasn't possible to cancel an exit-request executed by closing the main frame.
    This bug was found by Gareth Doutch (gdoutch).
    (Ticket 79)
  • Replaced all <layout> with <encoder> in logback*.xml.
    See Logback documentation for info.
  • Logback 0.9.21
  • SLF4J 1.6.0
  • commons-cli 1.2
  • httpclient 4.0.1
  • Groovy 1.7.2

Known Issues

  • Selection in the HTML view is currently somewhat buggy, especially in case of scaled view. This is a limitation of Flying Saucer that's used as the XHTML renderer.
  • cat and tail are currently still experimental features.
    AccessEvent isn't (and can't be) supported yet. LoggingEvent isn't fully supported, either.
