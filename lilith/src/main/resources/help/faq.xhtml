Frequently Asked Questions

Why is there no case-insensitive search?
Well, to be honest, this has two reasons:
  • Case-insensitive searching has quite a performance impact because every checked String has to be either lower- or uppercased.
    Beside that, lower- and uppercasing is Locale-dependent. This means that there is no way to do it reliably for every given unicode string.
  • I have simply no idea how to add that option in a way that is both userfriendly and applicable to the current condition implementation.
  • It's pretty easy to do it yourself - in the way expected by you - using groovy conditions.
Why is there no tree-view of the logger names as Chainsaw and Vigilog have?
One of the problems I'm facing is the (possibly) large amount of events I'm supporting.
This means that the logger names need to be saved upon reception separately from the events themselves. Otherwise all events would need scanning if an inactive logging files was reopened and this would take quite some time.
Something like that is in development but it doesn't have a very high priority right now for the following reasons:
  • I, personally, didn't use Chainsaws tree-view very much.
  • You can get a similar behavior using the filters already present in Lilith.
  • The tree-view itself would use valuable space on the screen. even if it could be toggled on and off.
  • I'm currently not in the mood to implement it just for "me too" reasons.
